# Dashboard KPIs - Comprehensive Analytics Implementation

## Overview
This document describes the comprehensive dashboard KPI implementation that fetches analytics data from the chatbot database based on date range filters.

## Implemented KPIs

### 1. Main Dashboard Metrics

#### Total Sessions
- **Calculation**: `COUNT(DISTINCT session_id)` from leads table
- **Description**: Count of distinct chat sessions in the filtered period
- **Implementation**: Uses `new Set()` to count unique session IDs

#### Total Unique Customers  
- **Calculation**: Count unique user identifiers from form data
- **Priority Order**: email → phone → user_id → session_id (fallback)
- **Description**: Distinct individuals who used the platform
- **Implementation**: Extracts identifiers from `form_data` field in leads

#### Average Session Time
- **Calculation**: `(end_time - start_time) / session_count` 
- **Source**: Message timestamps from `chat_history` array
- **Description**: Average time users spend per session (in minutes)
- **Implementation**: Calculates duration between first and last message timestamps
- **Validation**: Filters out invalid durations (>24 hours or ≤0)

#### Total Messages by Users
- **Calculation**: `COUNT(*) WHERE sender = 'user'` from chat history
- **Description**: Messages sent by customers
- **Implementation**: Counts messages with `sender: 'user'` in chat_history arrays

#### AI Responses
- **Calculation**: `COUNT(*) WHERE sender = 'bot'` from chat history  
- **Description**: Messages generated by AI system
- **Implementation**: Counts messages with `sender: 'bot'` in chat_history arrays

#### Total Leads
- **Calculation**: `COUNT(*)` from leads table in date range
- **Description**: Total lead records created in the period
- **Implementation**: Simple count of all leads matching date filter

### 2. Conversion Funnel Metrics

#### Sessions
- Same as Total Sessions above

#### Unique Customers
- Same as Total Unique Customers above  

#### User Messages Sent
- Same as Total Messages by Users above

#### AI Responses Generated
- Same as AI Responses above

#### Leads Captured
- **Calculation**: Count leads with non-empty form submissions
- **Filter**: `form_data` exists and has meaningful data (not just default values)
- **Description**: Users who submitted forms through Custom Forms or Adaptive Cards
- **Implementation**: Filters leads where `form_data` object has keys and values

## API Response Format

```json
{
  "totalSessions": number,
  "uniqueCustomers": number, 
  "avgSessionTime": number,
  "totalUserMessages": number,
  "totalAiResponses": number,
  "totalLeads": number,
  "funnel": {
    "sessions": number,
    "uniqueCustomers": number,
    "userMessages": number,
    "aiResponses": number,
    "leadsCaptured": number
  }
}
```

## Database Schema Used

### Leads Collection
```javascript
{
  _id: ObjectId,
  session_id: string,           // Used for session counting
  created_at: string,           // Used for date filtering
  chat_history: [               // Used for message analysis
    {
      id: string,
      sender: 'user' | 'bot',    // Used for message type counting
      message: string,
      timestamp: string          // Used for session duration calculation
    }
  ],
  form_data: {                  // Used for unique customer identification
    name?: string,
    email?: string,             // Primary identifier
    phone?: string,             // Secondary identifier
    company?: string,
    // ... other form fields
  },
  // ... other fields
}
```

## Date Range Filtering

- **Input Format**: ISO 8601 strings (`YYYY-MM-DDTHH:mm:ss.sssZ`)
- **Filter Applied**: `created_at` field in MongoDB queries
- **Default Behavior**: If no dates provided, returns all data
- **Query Structure**: `{ created_at: { $gte: startDate, $lte: endDate } }`

## Usage Examples

### API Calls
```bash
# Today's data
curl "http://localhost:3000/api/analytics?startDate=2025-11-25T00:00:00.000Z&endDate=2025-11-25T23:59:59.999Z"

# Last 30 days  
curl "http://localhost:3000/api/analytics?startDate=2025-10-26T00:00:00.000Z&endDate=2025-11-25T23:59:59.999Z"

# All data
curl "http://localhost:3000/api/analytics"
```

### Frontend Integration
```typescript
import { analyticsService } from '../services/analyticsService';

const dateRange = analyticsService.getDateRange('last30days');
const metrics = await analyticsService.getMetrics(dateRange);

// Access KPIs
console.log(`Total Sessions: ${metrics.totalSessions}`);
console.log(`Conversion Rate: ${(metrics.funnel.leadsCaptured / metrics.uniqueCustomers * 100).toFixed(1)}%`);
```
